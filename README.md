# StarBattlePlay — スターバトル 盤面チェックで遊ぶ（公開・静的アプリ）

スターバトル（星置きパズル）の盤面を、ブラウザでクリック/タップして手で解いて確かめる
**依存ゼロの静的アプリ**です。サーバ不要・オフライン動作。GitHub Pages で公開して、URL を
知っていれば誰でも遊べます。

## 中身

| ファイル | 役割 |
| --- | --- |
| `index.html` | 入口（ランチャー）。`board_play.html` へのリンク。 |
| `board_play.html` | アプリ本体。領域グリッドを描き、クリック/タップで 空→×→★→空。軽い「解けた」判定つき。 |
| `puzzles.js` | 盤面データ（`window.PUZZLES`）。領域グリッド＋各状態（`initial`/`fixpoint`/`forcing`/`solution`）。 |

`board_play.html` は `puzzles.js`（`window.PUZZLES`）があればそれを、無ければ内蔵フォールバックを使います。
データの更新は **`puzzles.js` を差し替えるだけ**（HTML は固定）。

## 遊び方

- サンプル一覧から盤面を選ぶ。
- プリセット状態（空 / 不動点 / フォーシング後 / 解）を読み込んで途中から確認。
  - 一部の大盤面（10x10）は**性能上「フォーシング後」状態を省略**しています（下記）。
- マスをクリック/タップで **空 → ×（星でない）→ ★（星）→ 空** に循環。
- 読み込んだ状態は 🔒 でロック（チェックボックスでロック中/解除を切替）。
- 各行/列/領域の★がちょうど k 個＆★が8近傍で非隣接になれば「✅ 解けました！」。

## ローカルで開く

`index.html` をブラウザで開くだけ（ビルド・インストール不要）。GitHub Pages では
リポジトリ Settings → Pages で `main` / `(root)` を有効化すると `…github.io/StarBattlePlay/` で開けます。

## データの由来と更新

盤面データ（`puzzles.js`）は**非公開リポのエクスポータが、公開対象として明示した盤面だけ**を
書き出したものです。連携は **非公開 → 公開の一方向**で、このリポジトリから非公開側へ戻る経路は
ありません。

- 状態の意味:
  - `initial` … 空盤（領域のみ）。
  - `fixpoint` … 名前付き技法 L1〜L4 の不動点（フォーシング直前）。
  - `forcing` … 不動点＋深さ1フォーシングで直に確定するマス。
  - `solution` … 真の解。
- **性能注意**: `forcing` の事前計算は 10x10 が非常に重い（約450秒/件）。このため **10x10 は
  `forcing` 状態を省略**し、`initial`/`fixpoint`/`solution` のみ収録しています。

このリポジトリは公開です。**入れたものは履歴も含め永久に公開**されます。

1. **トークン/秘密情報を絶対に入れない**（漏れると非公開リポが危険にさらされます）。
2. 公開してよいのは **盤面データ（領域＋状態）とアプリの HTML/JS のみ**。
   ソルバー/生成器のコード・全コーパス・内部ドキュメント（PLAN/SPEC 等）は**置きません**。
3. 盤面データは**公開してよいと明示されたものだけ**（非公開側のマニフェストが唯一の真実）。
   非公開リポ全体を同期する仕組みはありません。
4. 連携は **非公開 → 公開の一方向**。公開アプリが実行時に非公開リポを参照する仕掛け
   （トークン埋め込み等）は**作りません**。
